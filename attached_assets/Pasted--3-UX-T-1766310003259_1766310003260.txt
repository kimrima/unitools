🏗️ 3단계: 디자인 시스템 및 사용자 경험(UX) 최적화
이번 단계의 목표는 **"압도적인 편의성"**과 **"광고 최적화"**입니다. 디자인 때문에 로직이 깨지지 않도록 'Tailwind CSS'를 활용한 모듈형 디자인을 요청해야 합니다.

1. 전역 디자인 시스템 (Design Tokens) 설정
사이트 전체의 일관성을 위해 기본 컬러와 서체를 정의합니다.

메인 컬러: #3B82F6 (신뢰감을 주는 블루) 또는 #8B5CF6 (기술적인 느낌의 퍼플).

배경색: 기본 화이트 및 다크모드 대응을 위한 다크 그레이.

폰트: 글로벌 서비스이므로 영어와 한국어 모두 가독성이 좋은 Pretendard 또는 Inter.

요청사항: "Tailwind Config를 설정하여 전역에서 사용할 컬러 팔레트와 폰트 규격을 먼저 정의해 줘."

2. 메인 페이지(Home) 레이아웃 설계
수백 개의 도구를 사용자가 3초 안에 찾을 수 있어야 합니다.

Hero Section: 거대한 검색창(Search Bar)을 중앙에 배치. 사용자가 "PDF"라고 치면 실시간으로 관련 도구가 필터링되어야 함.

Category Grid: 8대 대분류를 카드 형태로 배치. 각 카드는 아이콘 + 타이틀 + 도구 개수(예: PDF Tools, 25개 도구) 포함.

Tool Cards: 개별 도구는 작고 직관적인 아이콘과 함께 리스트 형태로 노출.

3. 도구 상세 페이지(Tool Page) 공통 UI 구조
모든 도구(PDF 합치기, 이미지 압축 등)는 동일한 사용자 경험을 제공해야 합니다.

Step 1: 업로드 영역(Dropzone): 점선 테두리로 된 큰 영역. 파일 드래그 앤 드롭 지원 및 '파일 선택' 버튼.

Step 2: 설정 영역(Options): 파일 업로드 후 나타나는 세부 설정(예: PDF 순서 변경, 이미지 품질 선택).

Step 3: 처리 대기 화면 (광고 골든 타임): 이게 기획의 핵심입니다. 변환 버튼을 누르면 "처리 중(0~100%)" 애니메이션이 나오며, 그 주변에 광고가 들어갈 '빈 슬롯'을 미리 확보합니다.

Step 4: 완료 및 다운로드: 변환된 파일의 용량 변화(예: 5MB -> 1.2MB)를 보여주고 거대한 'Download' 버튼 제공.

4. 반응형 디자인 (Responsive)
모바일 환경에서도 파일 업로드가 막히지 않도록 버튼 크기와 간격을 충분히 확보합니다.

1. 디자인 기초 설정:

Tailwind.config.js를 설정하여 메인 블루 컬러(primary)와 다크모드 대응 컬러를 정의해 주세요.

전 세계 어디서나 가독성이 좋은 Inter 폰트를 적용해 주세요.

2. 공통 레이아웃 컴포넌트 제작:

Header: 로고, 8대 카테고리 메뉴, 다국어 선택기(Language Switcher) 포함.

Footer: 모든 도구 리스트를 텍스트 링크로 나열하는 SEO 최적화형 푸터.

Search Bar: 메인 페이지 중앙에 실시간 필터링이 가능한 도구 검색창 구현.

3. 통합 도구 UI 프레임워크 (The Tool Template):

모든 도구 페이지에 적용될 공통 UI를 만드세요.

파일 업로드 전(Dropzone) -> 업로드 후(Options) -> 처리 중(Progress with Ad Slot) -> 완료(Download)의 4단계 상태가 시각적으로 자연스럽게 전환되어야 합니다.

특히 '처리 중' 상태에서는 사용자가 지루하지 않게 애니메이션 바를 보여주고, 광고가 들어갈 수 있는 공간을 상/하단에 확보해 주세요.

4. 다국어 디자인 검수:

텍스트가 길어지는 언어(한국어, 스페인어 등)에서도 버튼이나 카드가 깨지지 않도록 유연한 레이아웃(Flex/Grid)을 사용해 주세요.
