📂 2단계 보강: 핵심 엔진 구축 및 8대 카테고리 로직 설계
개발팀에 **"우리는 지금 모든 도구의 '표준 규격'을 만드는 중이다"**라고 강조하며 아래 내용을 전달하세요.

1. 8대 카테고리별 핵심 엔진 라이브러리 셋팅
단순히 라이브러리 설치를 넘어, 각 카테고리의 '두뇌' 역할을 할 엔진들을 한꺼번에 셋팅합니다.

PDF 엔진 (pdf-lib, pdfjs-dist): 합치기, 나누기, 페이지 추출, 워터마크 삽입, PDF를 이미지로 렌더링하는 모든 로직의 기초.

이미지 엔진 (browser-image-compression, canvas): 리사이징, 압축, 포맷 변환(JPG/PNG/WebP/HEIC)의 기초.

비디오 엔진 (@ffmpeg/ffmpeg, @ffmpeg/util): 이게 핵심입니다. 비디오 자르기, MP4 to GIF, 오디오 추출 로직의 심장입니다.

텍스트 엔진 (tesseract.js): 이미지에서 글자를 읽어내는 OCR 로직의 기초.

2. 공통 처리 파이프라인 (The Pipeline) 설계
모든 도구는 다음의 5단계를 거칩니다. AI에게 이 **'표준 파이프라인'**을 코드로 짜달라고 하세요.

Input: Drag & Drop 또는 파일 선택 (Client-side Only).

Validation: 허용된 확장자인지, 파일 크기가 적절한지 체크.

Processing (핵심): * Wasm 또는 JS 라이브러리를 이용해 브라우저 메모리 내 연산.

이때 Progress Bar(0% -> 100%) 수치를 실시간으로 계산하는 로직 포함.

Preview: 처리된 결과물을 화면에 미리 보여주기 (URL.createObjectURL 활용).

Output: 최종 결과물을 파일명 설정(unitools_결과물.확장자)과 함께 다운로드.

3. 초기 핵심 도구 셋트 (테스트 그룹)
표준을 잡기 위해 아래 5개 도구를 **'기능적 프로토타입'**으로 먼저 구현합니다.

PDF: [PDF 합치기], [PDF 나누기]

이미지: [이미지 압축], [이미지 포맷 변환 (PNG to JPG)]

비디오: [비디오를 GIF로 변환] (FFmpeg.wasm 작동 여부 확인용)

💡 Replit AI에게 전달할 2단계 상세 지시 (수정본)
"수석 개발자님, 1단계 인프라 위에서 이제 UniTools의 '엔진 룸'을 설계하겠습니다. 이번 단계의 목표는 향후 200개 도구로 확장 가능한 '표준 로직 파이프라인'을 구축하는 것입니다."

1. 필수 라이브러리 통합 설치: pdf-lib, pdfjs-dist, browser-image-compression, @ffmpeg/ffmpeg, @ffmpeg/util, tesseract.js를 프로젝트에 추가해 주세요.

2. 공통 엔진 유틸리티 (/utils/engine) 작성:

File Processor: 모든 파일을 ArrayBuffer로 읽어 Wasm 엔진에 전달하는 공통 함수를 만드세요.

State Manager: 파일 처리 중 progress(0~100%), status(idle, processing, error, success)를 관리하는 커스텀 훅 useToolEngine을 작성해 주세요.

Worker Thread: 무거운 연산(비디오, PDF) 시 메인 화면이 멈추지 않도록 Web Worker 활용을 고려해 주세요.

3. 5대 핵심 테스트 도구 구현 (디자인 없이 로직만):

PDF: Merge(합치기) 및 Split(나누기) 로직.

Image: Compression(압축) 및 Converter(PNG<->JPG) 로직.

Video: MP4 to GIF 변환 로직 (FFmpeg.wasm 연결 확인).

4. 다국어 및 SEO 연동:

각 도구의 에러 메시지, 진행 상태 텍스트는 messages/ JSON에서 가져옵니다.

URL의 tool-id에 따라 해당 엔진이 로드되는 동적 스위칭 로직을 완성해 주세요.

"우리는 지금 200개 도구가 입주할 '아파트 뼈대'를 세우는 것입니다. 디자인은 신경 쓰지 말고, 이 5개 도구가 실제 브라우저 자원만으로 완벽하게 돌아가는지 코드로 증명해 주세요."
