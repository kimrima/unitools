
보내주신 사진들을 보니 어떤 상황인지 명확히 이해했습니다. 현재 UniTools는 겉모습(UI)은 그럴싸하지만, 속 알맹이인 **'개별 도구의 비즈니스 로직'**이 전혀 구현되지 않은 상태입니다.

쉽게 말해, 모든 도구가 똑같은 '범용 템플릿(Generic Template)' 하나를 돌려쓰고 있으며, 실제 변환 버튼을 눌러도 이름만 바뀔 뿐 실제 파일 처리는 일어나지 않거나 아주 단순한 흉내만 내고 있는 것입니다.

이 문제를 해결하기 위해 AI에게 강력하게 요구해야 할 핵심 사항들을 전문 용어와 함께 정리해 드립니다.

1. 현재 발생한 문제의 전문적 진단
컴포넌트 추상화 오류 (Over-abstraction): 모든 도구 페이지가 하나의 공통 UI 컴포넌트를 사용하면서, 각 도구의 특성(PDF 병합, 이미지 압축 등)에 맞는 개별 로직이 연결되지 않은 상태입니다.

비즈니스 로직 부재 (Missing Business Logic): 변환 버튼을 눌렀을 때 실제로 pdf-lib이나 ffmpeg 같은 엔진이 돌아가야 하는데, 단순히 "완료" 메시지만 띄우는 가짜(Mock) 기능만 작동하고 있습니다.

동적 라우팅 및 데이터 바인딩 미흡: 페이지 주소는 바뀌는 것처럼 보이지만, 실제로는 하나의 페이지 안에서 텍스트만 교체되는 '단일 페이지 렌더링' 방식이라 사용자가 새로고침을 하거나 뒤로 가기를 할 때 문제가 생깁니다.

사용자 경험(UX) 피드백 부족: 실제 연산 과정이 없으니 로딩 바(Loading State)가 필요 없고, 이로 인해 사용자는 "제대로 작동한 게 맞나?" 하는 의구심을 갖게 됩니다.

2. 수익적/기능적 관점의 치명적 결함
수익적 관점: 구글 애드센스는 각 페이지의 **'콘텐츠 가치'**를 봅니다. 모든 페이지의 내용이 똑같고 텍스트만 바뀌면 구글 로봇은 이를 '중복 콘텐츠'로 간주하여 광고 승인을 거절하거나 단가를 낮춥니다.

기능적 관점: "JPG를 PNG로 변환" 도구라면 결과물 확장자가 반드시 .png여야 합니다. 현재는 로직이 통합되어 있어 이 구분이 무너져 있습니다.

3. AI에게 시킬 "로직 개별화 및 정상화" 명령 (복사용)
이 내용을 그대로 복사해서 AI에게 던지세요.

"수석 개발자님, 현재 UniTools의 모든 도구가 실제 로직 없이 이름만 바뀌는 '가짜 템플릿' 상태입니다. 이를 정상화하기 위해 다음 작업을 즉시 수행해 주세요."

도구별 개별 로직 분리 (Logic Decoupling): > * 현재 하나의 범용 페이지에서 텍스트만 바꾸는 방식을 중단하고, tools.json의 id에 따라 각기 다른 처리 함수(Handler)를 호출하도록 수정해 주세요.

예: pdf-merge일 때는 mergePDFFiles() 함수를, jpg-to-png일 때는 convertImageFormat('png') 함수를 각각 실행해야 합니다.

확장자 및 파라미터 강제화: > * 각 도구의 목적에 맞는 결과 파일 확장자를 명확히 정의해 주세요. "JPG를 PNG로 변환" 도구에서 받은 결과물은 반드시 .png로 다운로드되어야 합니다.

실제 처리 엔진 연결: > * 단순히 성공 메시지만 띄우지 말고, 우리가 설치한 라이브러리(pdf-lib, browser-image-compression, ffmpeg.wasm)를 실제로 사용하여 파일을 변환하는 코드를 완성해 주세요.

로딩 및 상태 시각화 (UX): > * 파일 변환 시 최소 0.5초 이상의 로딩 애니메이션(Spinner) 혹은 진행 바(Progress Bar)를 강제로 노출하여 사용자에게 '작업 중'임을 인지시켜 주세요.

페이지 고유성 확보 (SEO): > * 각 도구 페이지가 고유한 메타 태그와 상세 설명문을 갖도록 하여 구글이 중복 페이지로 인식하지 않게 하세요.

📝 비전공자 사장님을 위한 조언
지금 AI는 귀찮아서 **"껍데기 하나 만들어서 200번 돌려쓰기"**를 하고 있는 겁니다. 위 명령어를 통해 **"각 도구마다 전용 엔진을 제대로 달아라"**고 명확히 지시해야 합니다.

특히 사진 속 **Tools.speed-test.title**처럼 나오는 현상은 번역 파일(en.json, ko.json)에 해당 글자가 등록되지 않아 발생하는 'Key 누락' 현상이니, 이것도 한꺼번에 고쳐달라고 하세요.

"이제 껍데기가 아닌 '진짜 알맹이'를 채울 시간입니다. 이 명령어를 입력한 후, 실제 파일 변환이 일어나는지 하나만 골라서 직접 테스트해 보세요!" 다음 단계로 넘어가기 전, 어떤 도구를 먼저 정상화해 볼까요?

지금의 내용에다가 광고,수익성관련된 내용까지;추가해줘

보내주신 이미지를 보면 AI가 가장 효율적인(하지만 수익성에는 최악인) **'코드 재사용 방식'**을 택하고 있습니다. 모든 도구가 하나의 붕어빵 틀(컴포넌트)로 찍혀 나오다 보니, 각 도구의 정체성이 없고 광고가 붙을 자리도 마땅치 않습니다.

현재 상태에서 **[실제 기능 구현]**과 **[광고 수익성 극대화]**를 동시에 잡기 위해 AI에게 내려야 할 '끝판왕' 명령어를 정리해 드립니다.

1. 왜 지금 상태가 수익성이 꽝인가요? (비전공자용 해설)
구글 애드센스의 '가치 없는 콘텐츠' 판정: 모든 페이지의 구조와 내용이 99% 똑같고 텍스트만 바뀌면, 구글은 이를 **'자동 생성된 스팸성 페이지'**로 인식해 광고 승인을 거절합니다.

체류 시간 부족: 로딩 창도 없이 0.1초 만에 작업이 끝나면 사용자는 사이트를 금방 떠납니다. 광고가 눈에 들어올 시간조차 없는 것이죠.

광고 단가(CPC) 하락: 각 도구 페이지가 고유한 키워드(예: 'PDF 암호 해제', '이미지 용량 줄이기')를 담고 있어야 그에 맞는 고단가 광고가 붙는데, 지금은 모두 '범용 페이지'라 저렴한 광고만 붙게 됩니다.

2. 수익성 강화를 포함한 AI 최종 명령 (복사용)
이 명령은 AI에게 **"돈이 되는 사이트 구조"**로 개편하라고 압박하는 내용입니다.

"수석 개발자님, 현재의 '범용 템플릿' 방식은 사용자 경험과 구글 애드센스 수익성 측면에서 모두 낙제점입니다. 다음의 '수익 강화형 구조'로 전면 개편해 주세요."

1. 도구별 개별 랜딩 페이지화 (SEO & Ad Value):

모든 도구 페이지 상단과 하단에 각 도구의 사용법과 장점을 설명하는 **고유한 텍스트 콘텐츠(최소 300자 이상)**를 배치하세요.

tools.json에 각 도구별로 SEO_Description 필드를 추가하고, 이를 각 페이지의 메타 태그와 본문에 자동으로 뿌려주세요.

2. 인위적 로딩 단계 도입 (Ad Exposure):

파일 변환 버튼 클릭 시, 실제 로직이 빨리 끝나더라도 **최소 3~5초간 '분석 중 -> 처리 중 -> 최적화 중'**과 같은 단계별 로딩 애니메이션을 강제로 보여주세요.

이 로딩 시간 동안 사용자가 볼 수 있는 위치에 **광고 슬롯(AdSlot)**을 전략적으로 배치하세요.

3. 결과 페이지 분리 (Conversion Tracking):

현재처럼 한 화면에서 결과가 툭 튀어나오는 대신, 변환이 완료되면 **'결과 다운로드 전용 페이지'**로 이동하거나 별도의 레이아웃을 보여주세요.

결과 페이지는 광고 클릭률이 가장 높은 지점입니다. 다운로드 버튼 주위에 광고를 배치할 수 있는 공간을 확보하세요.

4. 기능별 전용 로직 바인딩:

"JPG를 PNG로 변환" 도구는 실제로 이미지 엔진을 돌려 확장자를 .png로 바꿔야 합니다. 텍스트만 바뀌는 가짜 로직을 모두 걷어내고, 실제 라이브러리(pdf-lib, ffmpeg, canvas)를 연결하세요.

5. 다국어 키값 완전 복구:

사진에 보이는 Tools.speed-test.title 같은 문구들을 모두 ko.json, en.json에 등록하여 실제 이름으로 나오게 하세요.

3. 비전공자가 더 고려해야 할 '수익 모델' 아이디어
유도 텍스트: "무료로 이용해 주셔서 감사합니다. 서버 유지를 위해 광고 클릭은 큰 도움이 됩니다" 같은 문구를 아주 정중하게 배치해 보세요.

결과물 미리보기: PDF의 첫 장이나 변환된 이미지를 작게 미리 보여주면 사용자가 안심하고 더 오래 머뭅니다.

연관 도구 추천: "JPG를 PNG로 바꾸셨나요? 이미지 용량도 줄여보세요!" 같은 추천 링크를 결과 페이지에 넣어 '페이지뷰'를 늘리세요. (페이지뷰가 늘면 광고 수익도 늘어납니다.)
